# 医院信息管理系统 - 启动说明

## 快速启动指南

### 第一步：环境准备

确保已安装以下软件：
- **JDK 1.8或更高版本**
- **Maven 3.6或更高版本**
- **MySQL 8.0或更高版本**

### 第二步：数据库配置

1. **启动MySQL服务**

2. **创建数据库**
```bash
mysql -u root -p
```
然后执行：
```sql
source src/main/resources/db/schema.sql
```
或者直接执行：
```bash
mysql -u root -p < src/main/resources/db/schema.sql
```

3. **修改数据库配置**

编辑 `src/main/resources/application.yml` 文件，修改数据库连接信息：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hospital_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
```

### 第三步：编译项目

在项目根目录执行：
```bash
mvn clean install
```

### 第四步：运行项目

**方式一：使用Maven运行**
```bash
mvn spring-boot:run
```

**方式二：打包后运行**
```bash
mvn clean package
java -jar target/hospital-management-system-1.0.0.jar
```

**方式三：在IDE中运行**
- 打开 `HospitalManagementSystemApplication.java`
- 右键选择 "Run" 或 "Debug"

### 第五步：访问系统

1. 打开浏览器访问：`http://localhost:8080/hospital`

2. 使用默认账户登录：
   - **用户名**：admin
   - **密码**：admin123

## 常见问题

### 1. 数据库连接失败

**问题**：启动时报错 "Access denied" 或 "Unknown database"

**解决**：
- 检查MySQL服务是否启动
- 确认数据库用户名和密码是否正确
- 确认数据库 `hospital_db` 是否已创建

### 2. 端口被占用

**问题**：启动时报错 "Port 8080 is already in use"

**解决**：
- 修改 `application.yml` 中的 `server.port` 为其他端口（如8081）
- 或关闭占用8080端口的程序

### 3. 编译错误

**问题**：Maven编译失败

**解决**：
- 检查JDK版本是否为1.8+
- 检查Maven版本是否为3.6+
- 尝试删除 `~/.m2/repository` 后重新下载依赖

### 4. Lombok注解不生效

**问题**：IDE提示找不到getter/setter方法

**解决**：
- 安装Lombok插件（IntelliJ IDEA会自动提示安装）
- 在IDE中启用注解处理器

## 系统功能测试

登录成功后，可以测试以下功能：

1. **物资管理**
   - 添加物资：点击"添加物资"，填写信息后保存
   - 搜索物资：在搜索框输入关键词
   - 编辑物资：点击"编辑"按钮
   - 删除物资：点击"删除"按钮

2. **用户管理**
   - 查看用户列表
   - 添加新用户
   - 编辑用户信息
   - 删除用户

3. **库存管理**
   - 查看库存信息
   - 执行入库操作
   - 执行出库操作

## 开发模式

如果需要在开发模式下运行（支持热部署）：

1. 确保 `spring-boot-devtools` 依赖已添加（已在pom.xml中）
2. 在IDE中启用自动编译
3. 修改代码后会自动重启应用

## 生产部署

生产环境部署建议：

1. **修改默认密码**：首次登录后立即修改admin账户密码
2. **数据库安全**：使用强密码，限制数据库访问IP
3. **HTTPS配置**：配置SSL证书，使用HTTPS访问
4. **日志配置**：配置日志文件输出和轮转
5. **性能优化**：配置连接池、缓存等
