# 医院信息管理系统 - 项目文档

## 一、需求分析

### 1.1 项目背景

随着医院规模的不断扩大和医疗业务的日益复杂，传统的物资管理方式已经无法满足现代医院管理的需求。传统的手工记录方式存在以下问题：

- **效率低下**：手工记录和查询物资信息耗时耗力
- **信息不完整**：物资信息记录不完整，容易遗漏
- **库存管理困难**：缺乏实时库存信息，难以进行有效的库存控制
- **数据统计困难**：无法快速生成各类统计报表
- **权限管理缺失**：无法对不同用户进行权限控制

### 1.2 功能需求

#### 1.2.1 用户管理模块
- 用户注册与登录
- 用户信息管理（增删改查）
- 角色权限管理（管理员、普通用户）
- 用户状态管理（启用/禁用）

#### 1.2.2 物资管理模块
- 物资信息维护（增加、删除、修改、查询）
- 物资分类管理
- 物资编码管理
- 物资信息搜索功能

#### 1.2.3 库存管理模块
- 实时库存查询
- 入库管理（记录入库信息，更新库存）
- 出库管理（记录出库信息，更新库存）
- 库存预警（低库存提醒）
- 库存盘点功能

#### 1.2.4 供应商管理模块
- 供应商信息管理
- 供应商联系方式管理
- 采购记录关联

#### 1.2.5 统计报表模块
- 库存统计报表
- 出入库统计报表
- 物资使用情况统计
- 数据可视化展示

### 1.3 非功能需求

- **性能要求**：系统响应时间应在2秒以内
- **安全性要求**：用户密码加密存储，权限控制
- **可用性要求**：系统可用性达到99%以上
- **可维护性要求**：代码结构清晰，易于维护和扩展
- **用户体验要求**：界面简洁美观，操作便捷

## 二、设计思路

### 2.1 系统架构设计

本系统采用经典的**MVC（Model-View-Controller）架构模式**，结合Spring Boot框架实现：

```
┌─────────────────────────────────────────┐
│          Presentation Layer             │
│      (Thymeleaf Templates + HTML)       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│          Controller Layer                │
│    (Spring MVC Controllers)             │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│          Service Layer                   │
│    (Business Logic Services)             │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│          Repository Layer                │
│    (Spring Data JPA Repositories)       │
└─────────────────────────────────────────┘
                    ↓
┌─────────────────────────────────────────┐
│          Data Layer                      │
│         (MySQL Database)                 │
└─────────────────────────────────────────┘
```

### 2.2 数据库设计

采用关系型数据库MySQL，主要数据表包括：

- **users**：用户表
- **materials**：物资表
- **inventory**：库存表
- **stock_in**：入库记录表
- **stock_out**：出库记录表
- **suppliers**：供应商表
- **material_categories**：物资分类表

表之间通过外键关联，保证数据完整性。

### 2.3 模块化设计

系统采用模块化设计，主要模块包括：

1. **用户管理模块**：处理用户相关的所有业务逻辑
2. **物资管理模块**：处理物资信息的CRUD操作
3. **库存管理模块**：处理库存的增减和查询
4. **入库出库模块**：处理物资的入库和出库操作
5. **统计报表模块**：生成各类统计报表

### 2.4 安全设计

- 用户密码使用MD5加密存储（实际项目中应使用BCrypt等更安全的加密方式）
- 基于Session的用户认证机制
- 角色权限控制（管理员、普通用户）
- SQL注入防护（使用JPA参数化查询）

## 六、系统应用

### 6.1 系统部署

1. **环境要求**：
   - JDK 1.8+
   - Maven 3.6+
   - MySQL 8.0+

2. **部署步骤**：
   ```bash
   # 1. 创建数据库
   mysql -u root -p < src/main/resources/db/schema.sql
   
   # 2. 修改数据库配置
   # 编辑 src/main/resources/application.yml
   
   # 3. 编译项目
   mvn clean package
   
   # 4. 运行项目
   java -jar target/hospital-management-system-1.0.0.jar
   ```

3. **访问系统**：
   - 访问地址：http://localhost:8080/hospital
   - 默认管理员账户：admin / admin123

### 7.1 创新点

   - 卡片式布局，用户体验友好
   - 响应式设计，适配不同设备

2. **模块化架构**
   - 采用MVC架构，代码结构清晰
   - 模块化设计，易于维护和扩展
   - 分层明确，职责分明

3. **完善的业务逻辑**
   - 入库出库自动更新库存
   - 库存预警机制
   - 完整的用户权限管理

4. **良好的代码规范**
   - 使用Lombok简化代码
   - 统一的异常处理
   - 规范的命名和注释

### 7.2 不足与改进方向

1. **安全性方面**
   - ⚠️ 当前使用MD5加密，安全性较低
   - ✅ **改进**：应使用BCrypt等更安全的加密算法
   - ⚠️ 缺少CSRF防护
   - ✅ **改进**：集成Spring Security框架

2. **功能完善性**
   - ⚠️ 统计报表功能尚未完全实现
   - ✅ **改进**：完善报表功能，添加数据可视化图表
   - ⚠️ 缺少批量操作功能
   - ✅ **改进**：添加批量导入导出功能

3. **性能优化**
   - ⚠️ 未实现分页功能，大数据量时性能较差
   - ✅ **改进**：添加分页查询功能
   - ⚠️ 缺少缓存机制
   - ✅ **改进**：使用Redis缓存热点数据

4. **用户体验**
   - ⚠️ 缺少前端表单验证
   - ✅ **改进**：添加前端JavaScript验证
   - ⚠️ 缺少操作提示和确认对话框
   - ✅ **改进**：完善用户交互提示

5. **测试覆盖**
