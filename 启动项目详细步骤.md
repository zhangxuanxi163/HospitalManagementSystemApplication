# 医院信息管理系统 - 启动详细步骤

## 第一步：配置数据库

### 1. 确保MySQL服务已启动

- 检查MySQL服务是否运行
- 如果没有运行，启动MySQL服务

### 2. 创建数据库

打开MySQL命令行或MySQL Workbench，执行：

```sql
CREATE DATABASE IF NOT EXISTS hospital_db DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
```

或者直接执行SQL脚本：

```bash
# 在项目根目录执行
mysql -u root -p < src/main/resources/db/schema.sql
```

### 3. 修改数据库配置

编辑 `src/main/resources/application.yml` 文件，修改数据库连接信息：

```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/hospital_db?useUnicode=true&characterEncoding=utf8&useSSL=false&serverTimezone=Asia/Shanghai
    username: root        # 修改为你的MySQL用户名
    password: root        # 修改为你的MySQL密码
```

**重要**：请根据你的实际MySQL配置修改用户名和密码！

## 第二步：启动项目

### 方法一：在IntelliJ IDEA中启动（推荐）

1. **打开项目**
   - 如果项目已打开，跳过此步
   - 如果未打开：File → Open → 选择项目文件夹

2. **等待依赖下载完成**
   - 右下角会显示Maven依赖下载进度
   - 等待所有依赖下载完成

3. **找到主类**
   - 在项目树中找到：`src/main/java/com/hospital/HospitalManagementSystemApplication.java`

4. **运行项目**
   - **方式A**：右键 `HospitalManagementSystemApplication.java` → Run 'HospitalManagementSystemApplication'
   - **方式B**：点击类名旁边的绿色运行按钮（▶️）
   - **方式C**：使用快捷键：Shift + F10

5. **查看启动日志**
   - 在底部控制台查看启动日志
   - 看到 `Started HospitalManagementSystemApplication` 表示启动成功
   - 如果看到 `Tomcat started on port(s): 8080` 表示服务器已启动

### 方法二：在Eclipse中启动

1. **导入项目**
   - File → Import → Maven → Existing Maven Projects
   - 选择项目文件夹

2. **运行项目**
   - 找到 `HospitalManagementSystemApplication.java`
   - 右键 → Run As → Spring Boot App

### 方法三：使用Maven命令启动

**前提**：需要先安装Maven并配置环境变量

1. **打开命令行**
   - 在项目根目录打开PowerShell或CMD

2. **编译项目**
   ```bash
   mvn clean compile
   ```

3. **运行项目**
   ```bash
   mvn spring-boot:run
   ```

## 常见问题解决

### 问题1：数据库连接失败

**错误信息**：
```
com.mysql.cj.jdbc.exceptions.CommunicationsException: Communications link failure
```

**解决方法**：
1. 检查MySQL服务是否启动
2. 检查 `application.yml` 中的数据库配置是否正确
3. 检查数据库 `hospital_db` 是否已创建
4. 检查用户名和密码是否正确

### 问题2：端口被占用

**错误信息**：
```
Port 8080 is already in use
```

**解决方法**：
1. 修改 `application.yml` 中的端口：
   ```yaml
   server:
     port: 8081  # 改为其他端口
   ```
2. 或关闭占用8080端口的程序

### 问题3：依赖下载失败

**解决方法**：
1. 检查网络连接
2. 配置Maven镜像源（国内用户推荐）
3. 在IDE中重新加载Maven项目

### 问题4：编译错误

**解决方法**：
1. Build → Clean Project
2. Build → Rebuild Project
3. 检查Java版本是否为17
4. 检查IDE的Java SDK设置
